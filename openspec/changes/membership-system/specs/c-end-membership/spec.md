## ADDED Requirements

### Requirement: C端用户可以查看会员等级和权益
系统SHALL允许C端参赛者查看自己的会员等级、当前权益和有效期。

#### Scenario: 查看当前会员信息
- **WHEN** 用户访问个人中心的会员页面
- **THEN** 系统显示会员等级、有效期、可用权益列表

#### Scenario: 非会员用户查看
- **WHEN** 非会员用户访问会员页面
- **THEN** 系统显示会员介绍和开通入口

### Requirement: C端用户可以购买会员
系统SHALL支持C端用户通过支付购买会员服务。

#### Scenario: 选择会员套餐
- **WHEN** 用户选择会员套餐（月付/年付）
- **THEN** 系统显示套餐详情和价格

#### Scenario: 完成支付开通会员
- **WHEN** 用户完成支付
- **THEN** 系统立即开通会员并发送确认通知

#### Scenario: 支付失败
- **WHEN** 支付失败
- **THEN** 系统提示失败原因并保留订单30分钟

### Requirement: 会员等级体系
系统SHALL支持多级会员等级（普通会员、银卡会员、金卡会员、钻石会员）。

#### Scenario: 不同等级享有不同权益
- **WHEN** 用户升级到更高等级
- **THEN** 系统自动解锁对应等级的所有权益

#### Scenario: 会员降级
- **WHEN** 会员到期未续费
- **THEN** 系统将用户降级为普通会员并回收高级权益

### Requirement: 会员积分系统
系统SHALL为会员提供积分累积和消费功能。

#### Scenario: 参赛获得积分
- **WHEN** 会员完成赛事报名并参赛
- **THEN** 系统根据赛事类型和会员等级发放积分

#### Scenario: 积分抵扣
- **WHEN** 会员使用积分抵扣报名费
- **THEN** 系统按照积分兑换比例计算抵扣金额

#### Scenario: 积分过期
- **WHEN** 积分超过有效期（1年）
- **THEN** 系统自动清零过期积分并通知用户

### Requirement: 会员续费管理
系统SHALL支持会员自动续费和手动续费。

#### Scenario: 到期前提醒续费
- **WHEN** 会员到期前7天
- **THEN** 系统发送续费提醒（站内信+短信）

#### Scenario: 自动续费
- **WHEN** 用户开启自动续费且到期
- **THEN** 系统自动扣款并延长会员有效期

#### Scenario: 自动续费失败
- **WHEN** 自动续费扣款失败
- **THEN** 系统通知用户并保留3天宽限期

### Requirement: 会员权益使用记录
系统SHALL记录会员权益的使用情况。

#### Scenario: 查看权益使用历史
- **WHEN** 用户查看权益使用记录
- **THEN** 系统显示每项权益的使用时间、场景和剩余次数

#### Scenario: 权益用尽提醒
- **WHEN** 某项权益使用次数达到上限
- **THEN** 系统提示用户该权益已用尽并推荐升级
